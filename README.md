# Minor Project 3 – RedFlag: Fraud Detection Using SQL

# Project Overview

This project focuses on detecting fraudulent transaction patterns using SQL. The objective is to analyze transaction data and identify suspicious activities commonly observed in banking and financial systems.

By applying SQL queries, aggregate functions, joins, Common Table Expressions (CTEs), and window functions, the project uncovers multiple fraud scenarios that can help financial institutions detect and prevent fraudulent transactions.

---

## 🎯 Objectives

* Analyze transaction data using SQL.
* Detect suspicious transaction behaviors.
* Apply advanced SQL concepts to solve real-world fraud detection problems.
* Generate meaningful insights from financial transaction datasets.

## 🔍 Fraud Detection Scenarios

This project analyzes the following fraud patterns:

* Velocity Fraud Detection
* Round Amount Clustering
* Card Testing Detection
* Failed Then Successful Transactions
* Odd Hour Transaction Concentration
* Mule Account Detection
* Refund Abuse Detection
* Merchant Collusion Detection
* Just Under Threshold Transactions
* Dormant Then Active Accounts
* Velocity Spike Analysis
* Geographic Impossibility Detection
